
# ✨ GitHub 사용자 대시보드

이 프로젝트는 GitHub API를 활용하여 사용자의 프로필, 저장소, 프로젝트 정보를 시각적으로 보여주는 대화형 웹 대시보드입니다. 단일 사용자 정보를 깊이 있게 분석하거나, 두 사용자의 데이터를 나란히 비교하여 역량을 한눈에 파악할 수 있습니다. 다양한 차트와 시각화 요소를 통해 GitHub 활동을 직관적으로 이해할 수 있습니다.

실행주소1 : https://github-board-swart.vercel.app/

실행주소2 : https://dev-canvas-pi.vercel.app/

## 🚀 주요 기능

### 👤 1. 사용자 대시보드
- **프로필 요약**: 사용자의 프로필 사진, 이름, 자기소개, 팔로워/팔로잉 수 등 기본 정보를 표시합니다.
- **핵심 통계**: 공개 저장소, 팔로워, 팔로잉 수를 포함한 주요 통계를 모던한 카드로 시각화합니다.
- **기여 활동 히트맵**: 사용자의 GitHub 기여 활동을 시간에 따라 시각화하는 히트맵을 제공합니다.
- **시각화 차트**:
  - **언어 분포**: 사용자가 가장 많이 사용하는 프로그래밍 언어를 다채로운 **파이 차트**로 보여줍니다.
  - **인기 저장소 분석**: 가장 많은 별(star)을 받은 상위 5개 저장소의 별, 포크, 이슈 수를 **막대 차트**로 시각화합니다.
  - **저장소 크기 분포**: 상위 10개 저장소의 크기를 KB 단위로 **막대 차트**로 시각화합니다.
  - **포크 & 스타 비교**: 상위 5개 저장소의 포크와 스타 수를 비교하는 **막대 차트**를 제공합니다.
- **저장소 목록**:
    - 전체 저장소를 카드 형태로 나열합니다.
    - **검색, 언어 필터, 유형(소스/포크) 필터, 정렬(최신순, 별 개수순, 이름순)** 기능을 제공하여 원하는 저장소를 쉽게 찾을 수 있습니다.
    - **페이지네이션** 기능으로 20개씩 저장소를 볼 수 있습니다.
- **저장소 상세**: 저장소 카드를 클릭하면 해당 저장소의 GitHub 페이지가 새 탭에서 열립니다.
- **팔로워/팔로잉 모달**: 팔로워나 팔로잉 수를 클릭하면 상세 목록을 볼 수 있는 모달이 열립니다.
  - 목록에서 사용자를 클릭하여 해당 사용자의 GitHub 프로필로 이동할 수 있습니다.
  - "사용자 비교" 기능으로 현재 사용자와 선택한 팔로워/팔로잉을 비교할 수 있습니다.
- **프로젝트 목록**: 사용자가 참여 중인 공개 프로젝트 목록을 표시합니다.

### 👥 2. 사용자 비교
- **프로필 비교**: 두 사용자의 기본 프로필 정보와 핵심 통계를 나란히 비교합니다.
- **언어 분포 비교**: 두 사용자의 언어 사용 현황을 하나의 **통합 도넛 차트**로 시각화하여 기술 스택의 공통점과 차이점을 쉽게 파악할 수 있습니다.
- **역량 비교 분석**: 팔로워, 저장소 수, 총 스타 수 등 주요 지표를 **레이더 차트**로 시각화하여 두 사용자의 강점과 약점을 직관적으로 비교합니다.

### ✨ 3. 사용자 경험(UX)
- **검색 기록**: 최근 검색한 사용자를 기록하여 빠르게 다시 조회할 수 있습니다.
- **마지막 사용자 기억**: 앱을 다시 실행하면 마지막으로 조회했던 사용자의 정보를 자동으로 불러옵니다.
- **반응형 디자인**: 데스크톱, 태블릿, 모바일 등 다양한 기기에서 최적화된 화면을 제공합니다.
- **모던한 UI**: 그라데이션, 그림자 효과, 애니메이션 등을 활용한 현대적인 디자인을 제공합니다.
- **로딩 및 오류 처리**: 
  - 데이터를 불러오는 중에는 스피너를 표시합니다.
  - API 오류 발생 시 사용자 친화적인 메시지를 보여줍니다.
  - API 속도 제한 및 403 오류에 대한 상세한 안내와 자동 재시도 기능을 제공합니다.
- **GitHub 토큰 지원**: 
  - GitHub 개인 액세스 토큰(PAT)을 설정하여 API 요청 한도를 시간당 60회에서 5,000회로 늘릴 수 있습니다.
  - 토큰 설정 모달에서 토큰의 유효성을 즉시 검증합니다.

## 🛠️ 기술 스택

- **프레임워크**: React
- **언어**: TypeScript
- **스타일링**: Tailwind CSS (그라데이션, 애니메이션, 반응형 디자인)
- **차트 라이브러리**: Recharts (파이 차트, 막대 차트, 레이더 차트)
- **라우팅**: React Router DOM
- **API**: GitHub API (v3)
- **상태 관리**: React Hooks (useState, useEffect, useCallback, useMemo)
- **프록시 서버**: Express, http-proxy-middleware, cors

## ⚙️ 시작하는 방법

### 개발 모드 실행

1. 프로젝트를 클론합니다:
   ```bash
   git clone https://github.com/your-username/github-dashboard.git
   cd github-dashboard
   ```

2. 의존성 패키지를 설치합니다:
   ```bash
   npm install
   ```

3. 개발 서버를 실행합니다:
   ```bash
   npm run dev
   ```

4. 브라우저에서 `http://localhost:5173`으로 접속합니다.

### 프록시 서버 실행 (CORS 오류 해결)

GitHub API는 CORS 정책으로 인해 브라우저에서 직접 요청이 차단될 수 있습니다. 이 문제를 해결하기 위해 프록시 서버를 제공합니다:

1. 프록시 서버 실행:
   ```bash
   node server.js
   ```
   또는
   ```bash
   npm run proxy
   ```

2. 별도의 터미널에서 개발 서버 실행:
   ```bash
   npm run dev
   ```

3. 브라우저에서 `http://localhost:5173` 또는 Vite가 제공하는 URL로 접속합니다.

### 프로덕션 빌드

1. 프로젝트를 빌드합니다:
   ```bash
   npm run build
   ```

2. 빌드된 파일을 서비스합니다:
   ```bash
   npm start
   ```

3. 브라우저에서 `http://localhost:5173`으로 접속합니다.

### 대시보드 사용법
1. 검색창에 분석하고 싶은 GitHub 사용자명을 입력하고 '검색' 버튼을 클릭합니다.
2. 해당 사용자의 데이터가 화면에 표시됩니다.
3. 개요, 저장소, 프로젝트 탭을 클릭하여 다양한 정보를 확인할 수 있습니다.
4. 팔로워 또는 팔로잉 수를 클릭하면 상세 목록을 볼 수 있습니다.
5. 저장소 카드를 클릭하면 해당 GitHub 저장소 페이지가 새 탭에서 열립니다.

### 사용자 비교 사용법
1. 상단 내비게이션에서 '사용자 비교' 탭을 선택합니다.
2. 두 개의 입력창에 비교하고 싶은 GitHub 사용자명을 각각 입력하고 '비교' 버튼을 클릭합니다.
3. 두 사용자의 비교 데이터가 화면에 표시됩니다.
4. 팔로워 모달에서 '사용자 비교' 버튼을 클릭하여 현재 사용자와 선택한 팔로워를 비교할 수도 있습니다.

### GitHub 토큰 설정 방법
1. 상단 내비게이션에서 'API 토큰 설정' 버튼을 클릭합니다.
2. GitHub에서 발급받은 개인 액세스 토큰(PAT)을 입력합니다.
3. '저장' 버튼을 클릭하여 토큰을 저장합니다.
4. 토큰이 유효하면 API 요청 한도가 시간당 5,000회로 증가합니다.

## 🛡️ CORS 오류 해결 및 API 제한 관리

이 프로젝트는 GitHub API 사용 시 발생할 수 있는 문제를 해결하기 위한 여러 기능을 제공합니다:

### CORS 문제 해결
1. **개발 환경에서의 프록시 설정**
   - `vite.config.ts` 파일에 프록시 설정이 포함되어 있어 개발 서버에서 API 요청을 중계합니다.
   - `/api/github` 경로로 들어오는 요청을 `https://api.github.com`으로 리다이렉트합니다.

2. **독립 프록시 서버**
   - `server.js` 파일은 Express 기반의 프록시 서버를 구현합니다.
   - GitHub API 요청에 필요한 헤더를 자동으로 추가합니다.
   - CORS 헤더를 설정하여 브라우저에서의 API 요청을 허용합니다.
   - `node server.js` 또는 `npm run proxy` 명령으로 실행할 수 있습니다.

### API 속도 제한 관리
1. **GitHub 토큰 지원**
   - 개인 액세스 토큰(PAT)을 설정하여 API 요청 한도를 시간당 5,000회로 늘릴 수 있습니다.
   - 토큰은 브라우저의 로컬 스토리지에 안전하게 저장됩니다.
   - 토큰 유효성 검증 기능이 내장되어 있습니다.

2. **지능적인 요청 관리**
   - API 요청 큐 시스템으로 요청을 효율적으로 관리합니다.
   - 속도 제한에 도달하면 자동으로 요청 간격을 조정합니다.
   - 403 Forbidden 오류 발생 시 자동 재시도 메커니즘을 제공합니다.
   - 클라이언트 측 캐싱으로 중복 요청을 최소화합니다.
